About

This is a fully working clone for ESPNCricinfo mobile APP developed in React Native . Its a cross-platform app which works well for both Android and IOS and also having all the advanced features like Team and Series Search, Player Profiles, fastcast for blazingly fast scores updates, ball by ball commentary and many more. React Native is nothing but Native Components generated by JavaScript, so its much faster, smoother and gives better user experience than a conventional webview or a hybrid framework like PhoneGap, Xamarin. Choosing React Native over Native for mobile devlopment has lot of advantages which will be explained in further updates of docs .

Philosophy

This project was started in Jan-2018 with an aim to find a better replacement for webviews, but ended up developing a fully working clone of our Production ESPNCricinfo app.This is faster, smoother, lightweight and have webviews only for articles.

Major Packages Used

react @16.3.1

react-native @0.55.4

react-native-navigation ^1.1.436
React Native doesn't have inbuilt navigation modules, but community have some JS solutions, which are not smooth, so using React-Native-Navigation V1 which is a Native Package which supports both IOS and Android developed by Wix

react-native-tab-view
Tabbed Navigation is the trend today for both mobile and android, React Native community provides an excellent package for the same which uses Native View Pager for Android and ScrollView for IOS

react-native-video-player
This is a inhouse wrapper on top of React-Native-video package provided by the community

axios
React Native has inbuilt Native polyfill for xhr and supports fetch, but for the sake of better handling used Axios

base-64
Used to extract base64 messages during fastcast

create-react-context
App Supports theming (as of now only dark and light themes) which needs config objects to be passed across the component hierarchy, to avoid that we've used a higher order component connected to react context.

moment-timezone

redux State Management is a much needed feature for complex screens like cricket GamePackage, mainly in an app user can open the same screen multiple times on the stack which needs multiple state copies if we go for component based state, so Redux comes in rescue where it maintains only one state object per UID thereby improving memory leakage issues.

react-redux

redux-thunk

react-native-admob
This is an open source package used to support all our inApp Ads requirements like banners, Interstitial Ads, Rewarded Video Ads

react-native-code-push
This is a package developed by Microsoft with MIT licence inorder to support OTA (Over-the-Air) updates of the App, thereby bypassing the hassle of PlayStore procedures.
